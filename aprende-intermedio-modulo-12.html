Construir sobre Bittensor: APIs y SDK

El SDK Python de Bittensor te da acceso completo al estado de la red: consultar balances, leer el metagraph, hacer staking, firmar transacciones. Sin APIs propietarias, directamente on-chain.

Nivel Intermedio 6 horas Prerrequisito: Módulo I-11
// Objetivos del módulo
  • Instalar y configurar el SDK Python de Bittensor.
  • Usar las clases Subtensor, Metagraph y Wallet para consultar el estado de la red.
  • Ejecutar operaciones básicas: consultar saldo, hacer staking, leer el metagraph.
  • Entender la REST API del Worker de bittensor.pro y cómo usarla desde JavaScript.
  • Identificar proyectos prácticos que puedes construir sobre estas APIs.
// 01

El SDK Python de Bittensor

1.1. Instalación

El SDK oficial de Bittensor es una librería Python. Requiere Python 3.9 o superior. La instalación es estándar:

pip install bittensor

Esto instala todo lo necesario: conexión al nodo RPC, gestión de wallets (coldkey/hotkey), operaciones de staking, lectura del metagraph, y firma de transacciones. No hay dependencias de servicios externos: todo se conecta directamente al nodo público de Bittensor.

1.2. Las tres clases principales

  • Subtensor: la conexión al nodo. Te permite consultar el estado de la cadena, enviar transacciones, y leer cualquier dato on-chain.
  • Metagraph: el estado completo de una subnet en un momento dado. Arrays numpy con stake (S), incentive (I), trust (T), consensus (C), emisiones (E) para cada UID.
  • Wallet: gestión de coldkey y hotkey. Permite firmar transacciones y verificar identidad.
Concepto clave

A diferencia de Ethereum (donde necesitas un nodo propio o una API key de Infura/Alchemy), Bittensor expone un nodo RPC público en entrypoint-finney.opentensor.ai. El SDK lo usa por defecto. Para producción con alto volumen de consultas, considera montar tu propio nodo.

// 02

Conectarse y consultar estado

2.1. Conexión básica y consultas de red

import bittensor as bt

# Conectar al nodo principal
sub = bt.Subtensor(network="finney")

# Supply total emitido (en TAO, dividir entre 1e9 para RAO → TAO)
supply = sub.total_issuance().tao
print(f"Supply: {supply:.0f} TAO")

# Número de subnets activas
n_subnets = sub.get_total_subnets()
print(f"Subnets activas: {n_subnets}")

2.2. Consultar saldo de una dirección

address = "5GrwvaEF5zXb26Fz9rcQpDWS57CtERHpNehXCPcNoHGKutQY"
balance = sub.get_balance(address)
print(f"Saldo libre: {balance.tao:.4f} TAO")

2.3. Leer el metagraph de una subnet

import bittensor as bt
import numpy as np

mg = bt.metagraph(netuid=1) # Subnet 1

# Arrays de datos para cada UID
print(f"UIDs activos: {mg.n}")
print(f"Top 5 miners por incentive:")
top5 = np.argsort(mg.I)[-5:][::-1]
for uid in top5:
print(f" UID {uid}: incentive={mg.I[uid]:.4f}, stake={mg.S[uid]:.2f}")

Los arrays del metagraph son numpy arrays indexados por UID. mg.S = stake, mg.I = incentive, mg.R = rank, mg.T = trust, mg.E = emission, mg.C = consensus, mg.D = dividends.

// 03

Operaciones de wallet

3.1. Cargar y usar una wallet

wallet = bt.Wallet(name="mi_coldkey", hotkey="mi_hotkey")

# Ver las direcciones
print(f"Coldkey: {wallet.coldkeypub.ss58_address}")
print(f"Hotkey: {wallet.hotkey.ss58_address}")

3.2. Transferir TAO

dest = "5FHneW46xGXgs5mUiveU4sbTyGBzmstUspZC92UhjJM694ty"
amount = bt.Balance.from_tao(1.5) # 1.5 TAO

success = sub.transfer(
wallet=wallet,
dest=dest,
amount=amount
)
print(f"Transferencia: {'OK' if success else 'ERROR'}")

3.3. Hacer staking y unstaking

# Stakear 10 TAO en la hotkey de un validator
validator_hotkey = "5CFaLNiDfNBJCz5LoNMBFQNwXgSGzxBz5Q3Bj2PUiSGPyA6"
sub.add_stake(
wallet=wallet,
hotkey_ss58=validator_hotkey,
amount=bt.Balance.from_tao(10.0)
)

# Quitar todo el stake de ese validator
sub.unstake(
wallet=wallet,
hotkey_ss58=validator_hotkey,
amount=bt.Balance.from_tao(10.0)
)

Recuerda: estas operaciones requieren la coldkey desbloqueada. Nunca ejecutes estas operaciones en un servidor de minería donde solo debería estar la hotkey.

// 04

La REST API del Worker

4.1. Para proyectos JavaScript / sin Python

Si estás construyendo una aplicación web o no quieres usar Python, el Worker de bittensor.pro expone endpoints REST que encapsulan datos de Taostats con caché en memoria. Son los mismos endpoints que usa el frontend de bittensor.pro.

EndpointDatos que devuelveCaché
/api/price/taoPrecio TAO en USD, cambios 1h/24h/7d/30d30s
/api/subnet/listLista de todas las subnets con precio, MCap, TVL45s
/api/statsMétricas globales: supply, staking, subnets activas60s
/api/v1/account/balance/latest/v1Saldo libre y staking de una coldkey SS58Sin caché
/api/dtao/stake/latest/v1Posiciones de stake por subnet de una coldkeySin caché
// Ejemplo JavaScript: consultar precio de TAO
const res = await fetch(
'https://tao360.qrchd889gj.workers.dev/api/price/tao'
);
const data = await res.json();
console.log(`TAO: $${data.usd}`);

4.2. SDK vs REST API: cuándo usar cada una

CriterioSDK PythonREST API Worker
LenguajePython 3.9+Cualquier lenguaje (HTTP)
OperacionesLectura + escritura (staking, transfers)Solo lectura
LatenciaMedia-alta (RPC directo)Baja (caché en memoria)
LímitesNodo público con rate limit100K req/día (Workers free)
AutenticaciónNo requerida para lecturaNo requerida
Uso idealBots, scripts, minería, herramientas avanzadasDashboards web, apps ligeras
// 05

Ideas para construir

5.1. Proyectos de dificultad baja

  • Bot de alertas de precio en Telegram: Python script que comprueba el precio de TAO cada 5 minutos y envía un mensaje si supera un umbral. Usa el SDK o el Worker REST.
  • Dashboard de portfolio personal: script que lee el metagraph para tus hotkeys y calcula tu incentive acumulado por subnet.
  • Notificador de nuevas subnets: detecta cuándo aparece un netuid nuevo consultando la lista de subnets periódicamente.

5.2. Proyectos de dificultad media

  • Monitor de validators: alerta si el trust o los dividends de un validator específico caen por debajo de un umbral. Útil si tienes TAO delegado y quieres saber si tu validator está funcionando bien.
  • Comparador de APY en tiempo real: web app que consulta el metagraph de varias subnets y muestra los validators con mejor APY en cada una.
  • Tracker de emisiones por subnet: visualización histórica de cuántas emisiones ha recibido cada subnet a lo largo del tiempo, basada en el TVL histórico.

5.3. Buenas prácticas de desarrollo

Al construir herramientas sobre Bittensor, ten en cuenta: el nodo público tiene rate limits (no hagas cientos de peticiones por segundo). Usa caché local para datos que no cambian frecuentemente (el metagraph cambia cada tempo ~72 minutos). No hardcodees seed phrases ni claves privadas en el código — usa variables de entorno o ficheros de configuración fuera del repositorio. Si publicas tu herramienta, añade un disclaimer claro de que no es consejo financiero.

Glosario del módulo
SDK
Software Development Kit. Librería oficial que simplifica la integración con un protocolo. El SDK de Bittensor es la librería Python bittensor.
REST API
Interfaz HTTP que permite consultar datos mediante peticiones GET/POST. Los Workers de Cloudflare exponen una REST API sobre datos de Bittensor.
Clase Subtensor
Clase del SDK Python que representa la conexión al nodo Bittensor. Permite consultar balance, supply, subnets, y ejecutar transacciones.
Clase Metagraph
Clase del SDK que carga el estado completo de una subnet: arrays numpy con stake, incentive, trust, consensus y emisiones por UID.
Clase Wallet
Clase del SDK que gestiona coldkey y hotkey. Necesaria para firmar transacciones y autenticarse en la red.
RPC endpoint
Dirección del nodo al que conecta el SDK. Por defecto: entrypoint-finney.opentensor.ai. Acepta consultas WebSocket y JSON-RPC.
Rate limit
Límite de peticiones por unidad de tiempo. Los nodos públicos tienen rate limits para evitar abuso. Importante respetarlos en herramientas automáticas.
Balance.from_tao()
Función del SDK que convierte TAO (unidad legible) a RAO (unidad interna, 1 TAO = 10⁹ RAO). Necesaria para operaciones de transfer y staking.

  2. Usando el SDK, carga el metagraph de la subnet 1 (netuid=1). Imprime los 5 UIDs con mayor incentive y los 5 con mayor stake. ¿Coinciden los mismos UIDs en ambas listas?

  5. Diseña (en papel o pseudocódigo) un bot de alertas que notifique por Telegram si el precio de TAO supera los $500 o cae por debajo de $200. ¿Qué endpoint usarías? ¿Cada cuánto haría polling? ¿Cómo evitarías notificaciones duplicadas?
