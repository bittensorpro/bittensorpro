// Objetivos del módulo
  • Entender el proceso completo de creación de una subnet.
  • Aprender a usar el subnet template.
  • Diseñar un mecanismo de incentivos sólido.
  • Desplegar primero en localnet, luego testnet, luego mainnet.
Sección 13.2

¿Por qué crear una subnet?

Crear y mantener una subnet es un compromiso significativo. Las razones legítimas:

  • Tienes una idea de mercancía digital con demanda real (revenue potencial).
  • Posees expertise específico en un dominio que ninguna subnet cubre.
  • Quieres construir una empresa cuyo backend sea descentralizado.
  • Recibes el 18% de las emisiones de la subnet (incentivo económico significativo).

Pero también las contras:

  • Coste inicial alto (creación + capital para arrancar).
  • Mantenimiento continuo del código y la docs.
  • Riesgo de quedar deregistrado si el mecanismo no atrae flujo.
  • Necesidad de comunidad: sin mineros y validadores, no hay subnet.
Sección 13.3

Workflow general

  • Validar la idea: notebook Python con la lógica de tarea + scoring.
  • Adaptar al subnet template: github.com/opentensor/bittensor-subnet-template.
  • Probar en localnet: cadena local Subtensor en modo fast-blocks.
  • Probar en testnet: network test, capital de prueba sin valor real.
  • Crear en mainnet: registro en la red principal, requiere TAO real.
  • Atraer mineros y validadores: docs, Discord, comunicación.
  • Iterar y mantener: updates, parches, evolución del scoring.

Sección 13.5

Tutorial: subnet OCR (caso ejemplo)

La documentación oficial provee un tutorial completo de cómo convertir un notebook Python con código validado en una subnet funcional. El caso ejemplo es OCR (reconocimiento óptico de caracteres):

  • Tarea: extraer texto de imágenes de documentos.
  • Generación del challenge: el validador crea facturas sintéticas con la librería Faker, las convierte a PDF con ReportLab, y luego a imagen con ruido.
  • Synapse: OCRSynapse con campo image (input) y respuesta del minero.
  • Scoring: comparar el texto extraído con la verdad-suelo (los datos originales).
Sección 13.6

Diseñar el mecanismo de incentivos

El éxito o fracaso de tu subnet se decide aquí. Buenas prácticas:

  • Continuous improvement: evita scoring topado. Diseña funciones que recompensen mejoras incrementales.
  • Anti-gaming: seeds aleatorios, variaciones de input, validación de output completo.
  • Reproducibilidad relativa: usa thresholds, embeddings (CLIP), perceptual hashing si la salida no es exacta.
  • Velocidad vs calidad: balance entre latencia, fiabilidad y calidad.
  • Organic queries: integra peticiones reales de usuarios al proceso.
Sección 13.7

Despliegue paso a paso

13.7.1. Localnet

# Clonar Subtensor
git clone https://github.com/opentensor/subtensor.git
cd subtensor

# Compilar y arrancar en modo fast-blocks
BUILD_BINARY=1 ./scripts/localnet.sh

# En otra terminal, crear wallet de subnet owner
btcli wallet new-coldkey --wallet.name owner
btcli wallet new-hotkey --wallet.name owner --wallet.hotkey default

# Faucet de TAO en localnet
btcli wallet faucet --wallet.name owner --subtensor.network local

# Crear la subnet
btcli subnet create --wallet.name owner --subtensor.network local

13.7.2. Testnet

La red test usa endpoint --subtensor.network test. Necesitas TAO de testnet del faucet (limitado, suele ser competido).

13.7.3. Mainnet

Crear en mainnet implica:

  • Pagar el coste de creación (variable, históricamente entre 1.000 y 100.000 TAO según demanda).
  • Tener documentación pública sólida.
  • Tener al menos un minero y un validador iniciales (puedes ser tú mismo).

  1. Sigue el OCR Subnet Tutorial completo en docs.learnbittensor.org/tutorials/ocr-subnet-tutorial.
  2. Configura una localnet de Subtensor en tu máquina y crea una subnet de prueba.
  3. Despliega el subnet template en localnet con un minero y un validador. Verifica que las emisiones se mueven.
  4. Diseña en una página la propuesta de tu subnet ideal: tarea, scoring, hardware, USP frente a competencia, plan de mercado.
  5. Lee 3 READMEs de subnets reales (Templar SN3, Chutes SN64, Vidaio SN85) y compara enfoques.
  6. Reflexión: ¿qué subnet crearías tú? ¿Qué problema concreto resuelve? ¿Cómo demostrarás demanda real (revenue) en los primeros 6 meses?
