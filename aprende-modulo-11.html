// Objetivos del módulo
  • Elegir una subnet adecuada para minar.

Sección 11.2

¿Qué es minar en Bittensor?

Minar en Bittensor difiere significativamente de minar Bitcoin: es activo, creativo y competitivo. Prepararse para ser minero implica investigar la subnet adecuada para ti, dada tu experiencia y tu acceso a hardware.

Sección 11.3

Investigación previa

Antes de registrarte en una subnet, debes responder a:

  • ¿Qué tarea hace exactamente la subnet?
  • ¿Cuál es el modelo de scoring?
  • ¿Qué hardware se necesita?
  • ¿Cuál es el coste de registro actual? (varía dinámicamente)
  • ¿Cuántos UIDs hay disponibles? ¿Cuál es la tasa de desregistro?
  • ¿Cuál es el rendimiento esperado en alpha por día?

Recursos para investigar:

  • tao.app: dashboard de todas las subnets, con métricas en tiempo real.
  • github.com de la subnet: código, README, docs específicas.
  • Discord de Bittensor: canal de cada subnet con creadores y otros mineros.
  • Hyperparameters de la subnet (immunity_period crítico).
Sección 11.4

Coste y mecánica del registro

Registrarse cuesta TAO. El precio fluctúa dinámicamente en función del tiempo desde el último registro. Cuando aseguras un slot UID en una subnet en mainchain, este TAO es coste hundido y no se recupera. Una subnet puede tener un máximo de 64 validadores y 192 mineros (256 total) en subnets distintas a la 1.

btcli subnet register \
    --netuid <NETUID> \
    --wallet.name <COLDKEY> \
    --wallet.hotkey <HOTKEY> \
    --subtensor.network finney

Tras registrarse, tu hotkey se convierte en holder del UID slot. La hotkey puede tener múltiples UIDs en distintas subnets, pero solo un UID por subnet.

Sección 11.5

Ciclo de vida del minero

  • Te registras y obtienes un UID. Empieza tu immunity_period.
  • Publicas la IP:PORT de tu Axon en cadena.
  • Los validadores refrescan su metagraph y descubren tu Axon.
  • Los validadores envían peticiones y evalúan tus respuestas.
  • Acumulas emisiones desde cero durante el immunity_period.
  • Si al salir del immunity tus emisiones son las más bajas y no estás protegido, tu UID puede pasar al siguiente registrante (deregistro).
Sección 11.6

Hardware típico

ComponenteMínimo orientativoAlto rendimiento
GPU1 × RTX 3090 / 4090 (24 GB VRAM)1+ × H100/B200 (80 GB+)
RAM32 GB256 GB+ (validadores: 512 GB+)
Almacenamiento1 TB NVMe SSDMúltiples NVMe RAID
Red1 Gbps simétrica10 Gbps unmetered
CPUAMD Ryzen 9 / Intel i9EPYC / Xeon multi-socket

Subnets ligeras (predicción, scoring) pueden funcionar con hardware modesto. Subnets de inferencia LLM o entrenamiento requieren GPUs serias.

Sección 11.7

Despliegue básico

# Clonar la subnet
git clone https://github.com/<owner>/<subnet>.git
cd <subnet>

# Instalar dependencias
python3 -m venv venv
source venv/bin/activate
pip install -e .
pip install -r requirements.txt

# Ejecutar el minero
python neurons/miner.py \
    --netuid <NETUID> \
    --subtensor.network finney \
    --wallet.name <COLDKEY> \
    --wallet.hotkey <HOTKEY> \
    --axon.port 8901 \
    --logging.debug

Para producción, usa un process manager (pm2, systemd) y configura logs persistentes.

Sección 11.8

Errores típicos del principiante

ErrorCausaSolución
Mi minero no recibe queriesIP/port no anunciada o firewallVerificar serve_axon, abrir puerto
Emisión = 0 tras inmunidadMi código no compiteMejorar rendimiento, observar tops
DeregistradoBajas emisiones + slots llenosVolver a registrar, ajustar estrategia
Out of memory en GPUModelo demasiado grandeQuantization (4-bit, 8-bit) o GPU mayor
Validadores no me venTempo aún no completadoEsperar (~72 min) y refrescar metagraph
  1. Crea una hotkey de prueba y registra en testnet (red test) un UID en la subnet template oficial.
  2. Despliega el código miner.py de la plantilla y verifica que aparece en el metagraph.
  3. Investiga 3 subnets en mainnet en tao.app: anota coste de registro, hardware requerido y emisión esperada por UID.
  4. Calcula tu break-even: si registrarte cuesta X TAO y produces Y alpha/día con precio Z, ¿en cuántos días recuperas la inversión?
  5. Implementa logs personalizados en tu minero que registren la latencia de respuesta a cada validador.
  6. Reflexión: ¿qué subnet eligirías para empezar dado tu hardware actual? Justifica con datos.
